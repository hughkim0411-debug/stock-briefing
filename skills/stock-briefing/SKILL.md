# 증시 일일 브리핑 스킬

한국·미국 증시 정보를 두 개의 서브에이전트로 **병렬 수집**한 뒤, 교차 검증하고 초보자 친화적인 .docx 보고서로 만들어 작업 폴더에 저장합니다.

## 핵심 설계 원칙

1. **병렬 수집**: 한국경제(웹 스크래핑)와 네이버 뉴스(MCP 검색)는 서로 다른 도구를 쓰므로 동시에 실행하면 시간을 절반 가까이 단축할 수 있습니다.
2. **교차 검증**: 같은 사실이 두 소스에서 모두 확인되면 신뢰도 높음으로 처리. 한쪽에만 있으면 출처 명시.
3. **초보자 우선**: 모든 전문 용어는 풀어서 설명. 비유를 자주 사용.
4. **결과는 워킹 폴더에**: 항상 사용자의 현재 작업 폴더에 `.docx`로 저장.

## 실행 절차

### Step 1. 사용자 의도 확인 (옵션)

사용자가 명시적으로 정보를 모두 줬으면 건너뜁니다. 그렇지 않으면 `AskUserQuestion`으로 다음을 확인:

- 출력 형식: 채팅 요약만 / docx만 / 둘 다 (기본: 둘 다)
- 깊이: 핵심 위주 / 심층 분석 (기본: 심층)
- 카카오톡 전송: 예 / 아니오 (기본: 물어본 뒤 결정)

### Step 2. 두 서브에이전트 **병렬 실행** (★ 핵심)

반드시 **한 메시지에서 두 개의 Agent 호출을 동시에** 보내세요. 순차 호출 금지.

```
Agent({
  description: "한국경제 스크래핑",
  subagent_type: "hankyung-scraper",
  prompt: "오늘자 한국경제 hankyung.com 메인 페이지에서 한국·미국 증시 정보를 수집해주세요. 코스피/코스닥/미국 3대 지수/금리/유가/환율/주요 종목/전문가 코멘트를 추출해 구조화된 마크다운으로 반환하세요."
})

Agent({
  description: "네이버 뉴스 검색",
  subagent_type: "naver-news-searcher",
  prompt: "한국 시간 기준 오늘자 코스피·코스닥 마감, 뉴욕증시 마감, 원/달러 환율, 미국 국채금리 관련 최신 뉴스를 4개 쿼리로 병렬 검색해 구조화된 마크다운으로 반환하세요."
})
```

두 결과를 모두 받을 때까지 다음 단계로 진행하지 마세요.

### Step 3. 교차 검증

두 결과를 비교해 다음 표를 만드세요 (내부 검토용):

| 항목 | 한국경제 값 | 네이버 값 | 일치 여부 |
|---|---|---|---|
| 코스피 | ... | ... | ✓/✗/한쪽만 |
| 다우 | ... | ... | ✓/✗/한쪽만 |
| ... | | | |

불일치하는 항목이 있으면:
- 둘 다 같은 날짜인지 확인 (시차로 인한 차이 가능)
- 가장 최신 pubDate를 채택
- 보고서에 "두 소스 간 차이 있음" 명시

### Step 4. 보고서 생성

플러그인 내장 스크립트를 사용해 .docx 생성:

```bash
python3 "${CLAUDE_PLUGIN_ROOT}/skills/stock-briefing/scripts/build_report.py" \
  --data /tmp/briefing_data.json \
  --output "<워킹폴더>/증시브리핑_<YYYYMMDD>.docx"
```

스크립트가 받는 JSON 구조는 `scripts/build_report.py` 상단 주석 참조.

만약 스크립트가 없거나 실패하면, python-docx로 직접 작성합니다. 아래 "보고서 구성" 섹션을 따르세요.

### Step 5. 채팅 요약 작성

다음 구성으로 채팅에 응답:

1. **한 줄 요약** (가장 위)
2. **핵심 숫자** (표 형태): 코스피, 코스닥, 다우, S&P500, 나스닥, 환율, 미 30년물
3. **두 가지 핵심 원인** (불릿 2개)
4. **앞으로 어떻게 될까** (시나리오 3가지: 낙관/중립/비관, 확률 추정 포함)
5. **잠재 악재 6가지** (별점 ★~★★★★★)
6. **초보자 행동 지침 5가지**
7. **파일 링크**: `[보고서 보기](computer://<파일경로>)`
8. **Sources** 섹션: 두 소스의 URL 인용

### Step 6. (옵션) 카카오톡 전송

사용자가 카카오톡 전송을 요청하면, 다음 규칙을 따르세요:

- **메시지당 최대 500자** (안전 마진)
- 표시된 제한은 200자지만 실제는 약 880자에서 잘림 확인됨
- 긴 내용은 3개로 분할: (1/3) 요약+시장현황 / (2/3) 원인+시나리오+악재 / (3/3) 호재+행동지침+결론
- 도구: `mcp__KakaotalkChat-MemoChat`

## 보고서 구성 (build_report.py가 실패할 경우의 폴백)

다음 8개 장으로 구성:

1. **0장 — 꼭 알아야 할 7가지 단어**: 코스피/코스닥/다우/외국인/국채금리/반도체주/환율 풀이
2. **1장 — 한 문단 요약**: 색깔 박스 콜아웃
3. **2장 — 한국 증시**: 숫자 표 + 외국인 매도 설명 + 종목별 표
4. **3장 — 미국 증시**: 3대 지수 표 + 매그니피센트 7
5. **4장 — 두 가지 핵심 원인**: 금리, 반도체 (비유 콜아웃 포함)
6. **5장 — 오늘 새로 등장한 변수**: 환율, 파업, 지정학 등
7. **6장 — 전문가 시각**: 증권사 5곳 인용 표
8. **7장 — 앞으로 어떻게 될까**:
   - 7-1. 3가지 시나리오 (낙관/중립/비관)
   - 7-2. 악재 TOP 6 (별점)
   - 7-3. 호재 변수
   - 7-4. 초보자 행동 지침
   - 7-5. 한 줄 결론
9. **부록**: 용어 사전 + 자료 검증 메모

## 폰트 및 스타일

- 한글 폰트: `맑은 고딕`
- 본문: 11pt, line spacing 1.5
- 헤딩 1: 17pt, 색상 #1F3864
- 헤딩 2: 13pt, 색상 #2E74B5
- 콜아웃 박스: 배경 #E7F0F8 (정보), #FFF4D6 (결론), #E7F5E7 (행동 지침)

## 절대 하지 말 것

- ❌ 두 서브에이전트를 **순차** 호출하기 (반드시 병렬!)
- ❌ 추측으로 숫자 만들기 (모든 수치는 두 소스 중 하나에 명시되어야 함)
- ❌ 종목 매수/매도 직접 추천 (정보 제공만, 판단은 본인 책임 명시)
- ❌ 사용자 워킹 폴더가 아닌 임시 경로에 저장
- ❌ Word doc과 docx를 다른 것처럼 표현 (같은 것)
